Со Hub:
1 - Открываем GNS3
2 - Новый проект
3 - берём VPCS ставим 3 штуки
4 - Берём Ethernet Hub ставим его и соединяем все PC
5 - Задаём IP каждому PC (
PC1 = ip 192.168.10.11/24
PC2 = ip 192.168.10.22/24
PC3 = ip 192.168.10.33/24)

6 - Делаем Start Capture для PC2 - Hub
7 - выполнить ping PC1 → PC2 не менее четырёх раз, выполнить ping PC1 → PC3 не менее четырёх раз (ping 192.168.10.22/24 -c 4)

Со Switch:
Пункты 1-5 повторяются

6 - Делаем Start Capture для PC3 - Hub
7 - выполнить ping PC1 → PC2 и PC1 - PC3 (ping 192.168.10.22/24 -c 4)

3. Аналитический пункт по параметру C = 1

Нужно сравнить MAC-адрес назначения в ARP Request и ICMP Echo Request.

Сравнение
В ARP Request

MAC-адрес назначения:

ff:ff:ff:ff:ff:ff

Это широковещательный адрес broadcast.

В ICMP Echo Request

MAC-адрес назначения:

для PC1 -> PC2 в Hub: 00:50:79:66:68:01
для PC1 -> PC3 в Hub: 00:50:79:66:68:02
для PC1 -> PC3 в Switch: 00:50:79:66:68:02

Это уже обычный одноадресный unicast.

Пояснение

ARP Request отправляется как broadcast, потому что отправитель еще не знает MAC-адрес нужного узла и вынужден обращаться ко всем устройствам локального сегмента. После получения ARP Reply MAC-адрес становится известен, и дальнейшие кадры ICMP Echo Request передаются уже как unicast конкретному получателю.
