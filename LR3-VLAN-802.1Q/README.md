### Введение

>Когда в сети появляется несколько групп узлов, одной только физической схемы уже недостаточно. Нужно не просто соединить устройства проводами, а разделить трафик по логическим границам, чтобы широковещательные домены не смешивались между собой. Именно эту задачу решают VLAN. В данной лабораторной работе предстоит не настраивать сложный программный коммутатор с нуля, а посмотреть на результат его работы через Wireshark и по реальным кадрам доказать, как устроены trunk, access и тег 802.1Q.

## Цель работы

Целью лабораторной работы является изучение логического разделения коммутируемой сети на VLAN, понимание различия между access- и trunk-портами, а также практическое подтверждение того, что по trunk-соединению кадры передаются с тегом 802.1Q.

В результате выполнения лабораторной работы студент должен уметь:

1. объяснять, как коммутатор обучается MAC-адресам;
2. описывать, что такое flooding и когда он возникает;
3. объяснять назначение VLAN;
4. различать access-порт и trunk-порт;
5. читать схему стенда с двумя коммутаторами и четырьмя хостами;
6. находить в Wireshark кадры разных VLAN;
7. определять по кадру наличие тега 802.1Q и VLAN ID;
8. оформлять аналитический отчёт по готовому захвату трафика.

## Chapter II

### Теоретические основы лабораторной работы

#### Обучение коммутатора по MAC-адресам

Коммутатор канального уровня строит таблицу соответствий `MAC → порт`, наблюдая адреса источника входящих кадров. Когда коммутатор получает кадр, он запоминает, с какого порта пришёл адрес источника, и заносит эту информацию в MAC-таблицу.

Если коммутатор получает кадр и ещё не знает, на каком порту находится адрес назначения, он выполняет flooding — рассылает кадр по всем допустимым портам данного сегмента, кроме входного. После получения ответа от адресата MAC-таблица пополняется, и следующие кадры передаются уже адресно, а не широковещательно.

В рамках данной лабораторной работы это особенно важно для понимания ARP-запроса: именно широковещательный `ARP Request` сначала попадает в нужную VLAN, а уже затем `ARP Reply` возвращается как одноадресный кадр.

#### Зачем нужны VLAN

VLAN (`Virtual Local Area Network`) позволяет разделить один физический коммутатор на несколько логических широковещательных доменов. Узлы внутри одной VLAN могут обмениваться кадрами на втором уровне, а узлы разных VLAN не могут взаимодействовать напрямую на L2 без маршрутизации.

VLAN используется для:

- логического разделения пользователей и сервисов;
- уменьшения области широковещания;
- повышения управляемости сети;
- упрощения сегментации трафика.

Если два хоста физически подключены к одной коммутируемой инфраструктуре, это ещё не означает, что они принадлежат одному логическому сегменту. Принадлежность к VLAN важнее, чем сама физическая близость.

#### Access-порт и trunk-порт

Access-порт принадлежит одной VLAN и обычно подключается к конечному устройству. Конечный узел на таком порту получает и отправляет нетегированные Ethernet-кадры, а принадлежность кадра к VLAN задаётся самим коммутатором на основании конфигурации порта.

Trunk-порт переносит трафик нескольких VLAN одновременно. Для этого между коммутаторами в кадр добавляется специальный тег IEEE 802.1Q. Благодаря этому один физический линк может служить сразу нескольким логическим сегментам.

Именно поэтому при анализе трафика на trunk-соединении студент видит не «обычные» Ethernet-кадры, а кадры с добавленным полем VLAN.

#### Стандарт 802.1Q

При передаче кадра по trunk-соединению между MAC-адресами и полем `EtherType` вставляется тег 802.1Q. В этом теге особенно важно поле `VID` (`VLAN ID`), которое и указывает, к какой VLAN относится данный кадр.

Структура тега 802.1Q:

| Поле | Размер | Назначение |
|---|---|---|
| `TPID` | 2 байта | идентификатор тега (`0x8100`) |
| `PCP` | 3 бита | приоритет трафика |
| `DEI` | 1 бит | индикатор допустимости отбрасывания |
| `VID` | 12 бит | идентификатор VLAN |

Для данной лабораторной работы ключевыми являются два наблюдения:

1. у trunk-кадра присутствует `TPID = 0x8100`;
2. внутри тега можно явно прочитать `VLAN ID`.

#### Что необходимо понять по итогам теории

После изучения теоретического материала студент должен понимать:

1. как коммутатор обучается MAC-адресам;
2. почему широковещательные кадры ограничиваются рамками своей VLAN;
3. чем access-порт отличается от trunk-порта;
4. зачем между коммутаторами нужен trunk;
5. где именно в кадре появляется тег 802.1Q.

### Общие сведения о формате выполнения и опорной топологии

#### Формат выполнения без дополнительных установок

В облегчённой версии практикума лабораторная работа выполняется по готовому захвату, полученному с преднастроенного стенда. Такой формат позволяет проверить именно понимание VLAN и 802.1Q, не превращая работу в настройку Open vSwitch, Linux-коммутаторов или виртуализации на хостовой ОС.

Для обязательной части лабораторной достаточно:

- `GNS3` — для чтения и понимания схем стендов курса;
- `Wireshark` — для анализа файла захвата.

Дополнительные компоненты, такие как `Open vSwitch`, `GNS3 VM`, `Docker`, `VirtualBox` и `VMware`, для данной лабораторной **не требуются**.

#### Опорная топология

Логика стенда, по которому подготовлен захват:

```text
PC1 ---- SW1 ----\
                  \ 
                   == trunk ==
                  / 
PC2 ---- SW1 ----/           \---- SW2 ---- PC3
                              
                              \---- SW2 ---- PC4
```

Более точно распределение узлов по VLAN выглядит так:

- `PC1` подключён к `SW1` как access-порт `VLAN 10`;
- `PC2` подключён к `SW1` как access-порт `VLAN 20`;
- `PC3` подключён к `SW2` как access-порт `VLAN 10`;
- `PC4` подключён к `SW2` как access-порт `VLAN 20`;
- линк `SW1 — SW2` работает как `trunk` и переносит `VLAN 10` и `VLAN 20`.

Захват сделан **на trunk-линке** между `SW1` и `SW2`.

#### Таблица адресации

Для всех студентов используется единый фиксированный адресный план:

| Узел | IP-адрес | Роль |
|---|---|---|
| `PC1` | `192.168.10.11/24` | хост VLAN 10 на стороне `SW1` |
| `PC3` | `192.168.10.33/24` | хост VLAN 10 на стороне `SW2` |
| `PC2` | `192.168.20.22/24` | хост VLAN 20 на стороне `SW1` |
| `PC4` | `192.168.20.44/24` | хост VLAN 20 на стороне `SW2` |

#### Используемый файл захвата

В каталоге лабораторной должен находиться файл:

- `pcap/lr3_vlan_trunk_demo.pcapng`

В этом захвате представлены две отдельные короткие последовательности:

1. `ARP` + `ICMP` для `VLAN 10`;
2. `ARP` + `ICMP` для `VLAN 20`.

### Общие сведения о Wireshark

В данной лабораторной работе Wireshark используется для анализа уже готового файла захвата. Полезно заранее подготовить следующие фильтры отображения:

| Фильтр | Назначение |
|---|---|
| `vlan` | показать только кадры с тегом 802.1Q |
| `vlan.id == 10` | показать только кадры VLAN 10 |
| `vlan.id == 20` | показать только кадры VLAN 20 |
| `arp` | показать только ARP-кадры |
| `icmp` | показать только ICMP-пакеты |
| `arp && vlan.id == 10` | найти ARP внутри VLAN 10 |
| `icmp && vlan.id == 20` | найти ICMP внутри VLAN 20 |

При анализе обязательно проверь:

1. наличие поля `802.1Q Virtual LAN`;
2. значение `TPID = 0x8100`;
3. значение `VLAN ID`;
4. MAC- и IP-адреса отправителя и получателя.

### Задание 2. Анализ трафика VLAN 10 {#задание-2-анализ-трафика-vlan-10}
