## Chapter II

### Теоретические основы лабораторной работы

#### Обучение коммутатора по MAC-адресам

Коммутатор канального уровня строит таблицу соответствий `MAC → порт`, наблюдая адреса источника входящих кадров. Когда коммутатор получает кадр, он запоминает, с какого порта пришёл адрес источника, и заносит эту информацию в MAC-таблицу.

Если коммутатор получает кадр и ещё не знает, на каком порту находится адрес назначения, он выполняет flooding — рассылает кадр по всем допустимым портам сегмента, кроме входного. После получения ответа от адресата MAC-таблица пополняется, и последующие кадры направляются только на нужный порт.

Таким образом, процесс обучения коммутатора позволяет постепенно оптимизировать пересылку кадров и избегать ненужного flooding.

#### Зачем нужны VLAN

VLAN (`Virtual Local Area Network`) позволяет разделить один физический коммутатор на несколько логических широковещательных доменов. Узлы внутри одной VLAN могут обмениваться кадрами на L2, а узлы разных VLAN — нет, если между ними не включена маршрутизация на третьем уровне.

VLAN используется для:

- изоляции трафика между отделами или группами пользователей;
- сокращения области широковещания;
- повышения безопасности сети;
- упрощения управления сетевой инфраструктурой.

Каждая VLAN представляет собой отдельный широковещательный домен: broadcast-кадры, отправленные внутри одной VLAN, не выходят за её пределы.

#### Access-порт и trunk-порт

Access-порт принадлежит одной VLAN и обычно подключается к конечному устройству. Кадры, поступающие на access-порт, не тегируются — устройство на access-порту работает с нетегированным трафиком, а коммутатор сам назначает кадру VLAN ID в соответствии с конфигурацией порта.

Trunk-порт переносит трафик нескольких VLAN одновременно. Для идентификации принадлежности кадра к конкретной VLAN на trunk-порту используется тегирование по стандарту 802.1Q.

#### Стандарт 802.1Q

При передаче кадра по trunk-соединению между MAC-адресами и полем EtherType добавляется тег 802.1Q. Структура тега:

| Поле | Размер | Назначение |
|---|---|---|
| `TPID` | 2 байта | идентификатор протокола тега (`0x8100`) |
| `PCP` | 3 бита | приоритет кадра |
| `DEI` | 1 бит | индикатор допустимости отбрасывания |
| `VID` | 12 бит | идентификатор VLAN (VLAN ID) |

Поле VLAN ID позволяет принять решение о том, какому логическому сегменту принадлежит кадр. Именно это поле необходимо увидеть в Wireshark в ходе лабораторной работы.

#### Что необходимо понять по итогам теории

После изучения теоретического материала студент должен понимать:

1. как формируется MAC-таблица;
2. почему при неизвестном MAC выполняется flooding;
3. почему VLAN разделяет широковещательные домены;
4. зачем нужен trunk;
5. где именно в кадре появляется тег 802.1Q.

### Общие сведения о GNS3, Open vSwitch и Linux-узлах

#### Среда GNS3

В данной лабораторной работе используются:

- `SW1` и `SW2` — Linux-узлы с установленным Open vSwitch;
- `PC1`, `PC2`, `PC3`, `PC4` — VPCS в роли конечных хостов;
- Wireshark для анализа трафика на trunk-линке.

Linux-узлы SW1 и SW2 должны иметь по три сетевых адаптера каждый.

#### Почему используется Open vSwitch

Open vSwitch (`OVS`) позволяет явно задавать режимы портов, VLAN tags и trunk без необходимости импортировать проприетарные образы коммутаторов. Это делает лабораторную работу воспроизводимой в учебной среде.

Основные команды OVS, необходимые для данной работы:

| Команда | Назначение |
|---|---|
| `apt update && apt install -y openvswitch-switch` | установка OVS |
| `ovs-vsctl add-br br0` | создание bridge |
| `ovs-vsctl add-port br0 eth0` | добавление порта в bridge |
| `ovs-vsctl set port eth1 tag=10` | назначение access-порта VLAN 10 |
| `ovs-vsctl set port eth2 tag=20` | назначение access-порта VLAN 20 |
| `ovs-vsctl set port eth0 trunks=10,20` | настройка trunk с разрешёнными VLAN |
| `ovs-vsctl show` | отображение текущей конфигурации |

#### Команды настройки сети на Linux-узлах

Для настройки IP-адресов на VPCS:

| Команда | Назначение |
|---|---|
| `ip <адрес>/<маска>` | назначить IP-адрес и маску |
| `ping <IP-адрес>` | проверить связность |
| `save` | сохранить конфигурацию |

### Задание 1. Сборка стенда и настройка VLAN

Для этого задания используется параметр `A`, который вычисляется по формуле:

- `A = ((N - 1) mod 4) + 1`

Вариант адресного плана по параметру `A`:

| `A` | `PC1` | `PC2` | `PC3` | `PC4` |
|---|---|---|---|---|
| 1 | `10.10.10.10/24` (VLAN10) | `10.20.20.20/24` (VLAN20) | `10.10.10.30/24` (VLAN10) | `10.20.20.40/24` (VLAN20) |
| 2 | `172.16.10.10/24` (VLAN10) | `172.16.20.20/24` (VLAN20) | `172.16.10.30/24` (VLAN10) | `172.16.20.40/24` (VLAN20) |
| 3 | `192.168.10.10/24` (VLAN10) | `192.168.20.20/24` (VLAN20) | `192.168.10.30/24` (VLAN10) | `192.168.20.40/24` (VLAN20) |
| 4 | `10.0.10.10/24` (VLAN10) | `10.0.20.20/24` (VLAN20) | `10.0.10.30/24` (VLAN10) | `10.0.20.40/24` (VLAN20) |

Для лабораторной работы используются два коммутатора `SW1` и `SW2` и четыре конечных узла `PC1`, `PC2`, `PC3`, `PC4`. Распределение VLAN одинаково для всех вариантов: `PC1` и `PC3` — VLAN10, `PC2` и `PC4` — VLAN20. Такое распределение нужно для того, чтобы одна и та же VLAN находилась по обе стороны trunk-соединения.

Топология стенда:

Порядок настройки OVS на `SW1` и `SW2`:

1. установить `openvswitch-switch`;
2. создать bridge `br0`;
3. добавить физические интерфейсы в bridge;
4. настроить access-порты под VLAN10 и VLAN20;
5. настроить trunk между `SW1` и `SW2`, разрешив VLAN10 и VLAN20.
