### Введение

>Физическая избыточность в сети полезна ровно до того момента, пока не превращается в бесконечное хождение кадров по кругу. Если несколько коммутаторов соединены с образованием цикла, широковещательные кадры начинают размножаться, таблицы MAC-адресов становятся нестабильными, а сеть быстро перестаёт работать нормально. Чтобы этого не происходило, применяется Spanning Tree Protocol. В данной лабораторной работе нужно не настраивать STP на сложном программном коммутаторе, а посмотреть на его работу через Wireshark и по реальным кадрам понять, как сеть удерживается от шторма и как резервный путь становится активным после отказа основного линка.

## Цель работы

Целью лабораторной работы является практическое изучение проблемы петель на втором уровне модели OSI, понимание назначения STP и BPDU, а также наблюдение того, как резервный линк начинает использоваться после изменения топологии.

В результате выполнения лабораторной работы студент должен уметь:

1. объяснять, почему L2-петли опасны;
2. описывать назначение STP;
3. объяснять роль BPDU в расчёте связующего дерева;
4. читать схему кольцевой топологии;
5. находить в захвате служебные STP-кадры;
6. определять по полям кадра корневой коммутатор;
7. объяснять, почему на резервном линке сначала видны только BPDU, а затем появляется пользовательский трафик;
8. оформлять аналитический отчёт по готовому захвату.

## Chapter II

### Теоретические основы лабораторной работы

#### Что такое петля на канальном уровне

Если несколько коммутаторов образуют цикл, широковещательные кадры могут бесконечно циркулировать по сети. На канальном уровне нет поля TTL, поэтому кадр сам по себе не «затухает». Это приводит к ряду проблем:

- к лавинообразному росту широковещательного трафика;
- к дублированию кадров;
- к нестабильности MAC-таблиц;
- к перегрузке коммутаторов и каналов связи.

Именно поэтому простое физическое соединение коммутаторов «в кольцо» без управляющего механизма является опасным.

#### Назначение STP

Spanning Tree Protocol (`STP`, IEEE 802.1D) строит логическую топологию без петель поверх физически избыточной схемы. Для этого протокол:

1. выбирает корневой коммутатор (`Root Bridge`);
2. рассчитывает стоимость путей до корня;
3. оставляет активными только те порты, которые необходимы для построения дерева;
4. переводит избыточные порты в заблокированное состояние.

В результате физическое кольцо остаётся на месте, но логически сеть работает как дерево без циклов.

#### Роль BPDU

BPDU (`Bridge Protocol Data Unit`) — это служебные кадры, которыми коммутаторы обмениваются для расчёта связующего дерева. По этим кадрам можно определить:

- какой коммутатор считается корневым;
- кто именно отправил данный BPDU;
- через какой порт он был отправлен;
- какая стоимость пути до корня считается лучшей.

Для лабораторной работы особенно важны следующие поля:

| Поле | Назначение |
|---|---|
| `Root ID` | идентификатор корневого коммутатора |
| `Bridge ID` | идентификатор отправителя |
| `Port ID` | идентификатор порта отправки |
| `Root Path Cost` | стоимость пути до корневого коммутатора |

#### Почему резервный линк важен

Избыточный линк в кольце нужен не для того, чтобы всегда передавать пользовательские кадры по всем путям сразу, а для того, чтобы пережить отказ одного из основных соединений. Пока сеть находится в устойчивом состоянии, резервный путь может быть логически заблокирован STP. После изменения топологии ранее заблокированный линк становится полезным и начинает участвовать в передаче данных.

Именно это и предстоит увидеть в данной лабораторной работе: на одном и том же линке сначала видны только BPDU, а затем — обычный `ARP` и `ICMP`.

#### Что необходимо понять по итогам теории

После изучения теоретического материала студент должен понимать:

1. почему петли на L2 опасны;
2. какую задачу решает STP;
3. зачем коммутаторы обмениваются BPDU;
4. как по кадрам BPDU определить корневой коммутатор;
5. почему резервный линк может быть неактивен в обычном режиме и активироваться после отказа.

### Общие сведения о формате выполнения и опорной топологии

#### Формат выполнения без дополнительных установок

В облегчённой версии практикума лабораторная работа выполняется по готовому захвату, подготовленному на заранее настроенном стенде. Такой формат позволяет проверить понимание STP, BPDU и поведения сети при изменении топологии без требования вручную поднимать `Open vSwitch`, Linux-коммутаторы и внешнюю виртуализацию.

Для обязательной части лабораторной достаточно:

- `GNS3` — для чтения схем стендов курса;
- `Wireshark` — для анализа файла захвата.

Дополнительные компоненты, такие как `Open vSwitch`, `GNS3 VM`, `Docker`, `VirtualBox` и `VMware`, для данной лабораторной **не требуются**.

#### Опорная топология

Захват подготовлен по следующей логике:

```text
           SW2
          /   \
         /     \
       SW1-----SW3
        |       |
       PC1     PC2
```

- `SW1`, `SW2`, `SW3` образуют физическое кольцо;
- `PC1` подключён к `SW1`;
- `PC2` подключён к `SW3`;
- захват выполнен на линке `SW1 — SW3`.

В интерпретации данной лабораторной этот линк рассматривается как **резервный**:

- в устойчивом состоянии на нём видны служебные кадры `BPDU`;
- после отказа активного пути через `SW2` STP перестраивает дерево;
- на том же самом линке появляется пользовательский трафик.

#### Адреса и идентификаторы, используемые в захвате

IP-адреса конечных хостов:

| Узел | IP-адрес |
|---|---|
| `PC1` | `10.0.0.11/24` |
| `PC2` | `10.0.0.22/24` |

MAC-адреса коммутаторов, видимые в BPDU:

| Коммутатор | MAC-адрес |
|---|---|
| `SW1` | `02:42:ac:00:00:01` |
| `SW3` | `02:42:ac:00:00:03` |

Корневым коммутатором в данном стенде считается `SW1`.

#### Используемый файл захвата

В каталоге лабораторной должен находиться файл:

- `pcap/lr4_stp_bpdu_demo.pcapng`

Внутри файла представлены две фазы:

1. устойчивое состояние — на резервном линке присутствуют только служебные STP-кадры;
2. изменение топологии и восстановление передачи данных — на том же линке появляется пользовательский трафик.

### Общие сведения о Wireshark

Полезные фильтры отображения для данной лабораторной работы:

| Фильтр | Назначение |
