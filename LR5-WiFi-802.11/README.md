## Chapter II

### Теоретические основы лабораторной работы

#### Особенности беспроводной среды

В проводной Ethernet-сети узлы соединены кабелем и передают кадры по физически отделённым каналам. В Wi-Fi передача выполняется через общую радиосреду. Из-за этого возрастает вероятность коллизий, скрытых узлов и интерференции.

Основные отличия беспроводной среды от проводной:

- все станции разделяют один канал связи;
- невозможно обнаружить коллизию во время передачи;
- существуют скрытые станции — узлы, которые не слышат друг друга, но при этом мешают общему получателю;
- сигнал затухает с расстоянием и подвержен помехам.

Эти особенности требуют специальных механизмов управления доступом к среде.

#### CSMA/CA

В IEEE 802.11 применяется механизм CSMA/CA (`Carrier Sense Multiple Access with Collision Avoidance`). В отличие от CSMA/CD в классическом Ethernet, где коллизия обнаруживается постфактум, в Wi-Fi станции стараются избежать коллизий заранее.

Принцип работы CSMA/CA:

1. перед отправкой станция «слушает» эфир;
2. если канал свободен в течение определённого времени (DIFS), станция начинает передачу;
3. если канал занят, станция ожидает и использует backoff-механизм — случайную задержку перед повторной попыткой;
4. после успешного получения кадра получатель отправляет подтверждение ACK.

Подтверждения ACK являются обязательной частью Wi-Fi, поскольку отправитель не может определить, произошла ли коллизия, по аналогии с проводным Ethernet.

#### Классы кадров 802.11

Кадры Wi-Fi делятся на три основные группы:

1. **Management-кадры** — кадры управления и присоединения к сети:
   - `Beacon` — рассылается точкой доступа для объявления о наличии сети;
   - `Authentication` — кадр аутентификации клиента;
   - `Association` — кадр ассоциации клиента с точкой доступа;
   - `Probe Request / Probe Response` — кадры активного сканирования.

2. **Control-кадры** — служебные кадры управления передачей:
   - `ACK` — подтверждение получения кадра;
   - `RTS` (`Request to Send`) — запрос на отправку;
   - `CTS` (`Clear to Send`) — разрешение на отправку.

3. **Data-кадры** — кадры полезной нагрузки.

Тип и подтип кадра определяются полями `Frame Control` в заголовке 802.11. Значения type/subtype:

| Type | Подтип | Кадр |
|---|---|---|
| `0` (management) | `0x08` | Beacon |
| `0` (management) | `0x0b` | Authentication |
| `0` (management) | `0x00` | Association Request |
| `0` (management) | `0x01` | Association Response |
| `1` (control) | `0x1d` | ACK |
| `1` (control) | `0x1b` | RTS |
| `1` (control) | `0x1c` | CTS |
| `2` (data) | различные | Data |

#### Этапы подключения клиента к точке доступа

Упрощённо подключение клиента к Wi-Fi сети можно представить следующим образом:

1. **Beacon** — клиент обнаруживает сеть по периодическим Beacon-кадрам, рассылаемым точкой доступа;
2. **Authentication** — выполняется аутентификация (обмен Authentication-кадрами);
3. **Association** — клиент ассоциируется с точкой доступа (обмен Association Request/Response);
4. **Data** — начинается обмен данными;
5. **ACK** — переданные кадры подтверждаются ACK.

Именно эту последовательность предстоит увидеть и описать в захваченном дампе трафика.

#### RTS/CTS

RTS/CTS — дополнительный механизм уменьшения вероятности коллизий, особенно в условиях скрытых станций. Принцип работы:

1. перед основной передачей отправитель посылает короткий кадр **RTS** (`Request to Send`);
2. принимающая сторона отвечает кадром **CTS** (`Clear to Send`);
3. после этого остальные станции видят, что эфир занят (по полю Duration в RTS/CTS), и воздерживаются от передачи;
4. основная передача и ACK завершают обмен.

RTS/CTS не используется для каждого кадра, но его наличие в дампе подтверждает, что механизм CSMA/CA может включать этот дополнительный шаг.

#### Адресация в кадрах 802.11

В кадрах Wi-Fi адресация устроена сложнее, чем в Ethernet. В заголовке 802.11 могут присутствовать до четырёх адресных полей:

| Поле | Назначение |
|---|---|
| `Address 1` | адрес получателя кадра |
| `Address 2` | адрес отправителя кадра |
| `Address 3` | адрес фильтрации (обычно BSSID) |
| `Address 4` | используется только в WDS-режиме |

Интерпретация адресов зависит от значений флагов `To DS` и `From DS`:

| To DS | From DS | Address 1 | Address 2 | Address 3 | Address 4 |
|---|---|---|---|---|---|
| 0 | 0 | Destination | Source | BSSID | — |
| 0 | 1 | Destination | BSSID | Source | — |
| 1 | 0 | BSSID | Source | Destination | — |
| 1 | 1 | Receiver | Transmitter | Destination | Source |

Для анализа data-кадра необходимо определить значения `To DS` и `From DS` и на их основе установить роли каждого адреса.
