## Chapter II

### Теоретические основы лабораторной работы

#### Статическая маршрутизация

Маршрутизатор принимает решение о пересылке пакета на основании таблицы маршрутов. Таблица маршрутов содержит записи, каждая из которых указывает: для какой сети (адрес назначения и маска) через какой шлюз (next hop) или через какой интерфейс следует пересылать пакет.

В статической маршрутизации все нужные записи добавляются вручную. Это отличается от динамической маршрутизации, где маршруты обновляются автоматически протоколами вроде OSPF или BGP.

Для успешной маршрутизации необходимо:

1. чтобы на каждом маршрутизаторе был маршрут к сети назначения (или маршрут по умолчанию);
2. чтобы на всех промежуточных маршрутизаторах был включён IP forwarding;
3. чтобы обратный маршрут также существовал (пакет должен вернуться).

#### Longest prefix match

Если для одного и того же адреса назначения в таблице маршрутов подходят несколько записей, выбирается запись с самым длинным префиксом, то есть наиболее специфичный маршрут.

Например, если в таблице одновременно присутствуют:

- `10.20.0.0/16` через `10.0.12.2`
- `10.20.20.0/24` через `10.0.12.2`

то для адреса `10.20.20.10` будет выбран маршрут `/24`, поскольку он является более специфичным. Если маршрут `/24` удалить, тот же адрес будет направлен по маршруту `/16`.

Это правило гарантирует, что более точные маршруты всегда имеют приоритет над более общими.

#### Поле TTL

TTL (`Time to Live`) — это счётчик переходов пакета через маршрутизаторы. На каждом маршрутизаторе значение TTL уменьшается на единицу. Когда TTL достигает нуля, пакет отбрасывается, а отправителю отправляется ICMP-сообщение `Time Exceeded`.

TTL выполняет две функции:

1. **защита от петель маршрутизации** — даже при наличии петли пакет не будет циркулировать бесконечно;
2. **диагностика** — по значению TTL можно определить, через сколько маршрутизаторов прошёл пакет.

Например, если исходный TTL равен 64 (типичное значение для Linux), а в захваченном на транзитном линке пакете TTL равен 62, это означает, что пакет прошёл через 2 маршрутизатора.

#### MTU и фрагментация

MTU (`Maximum Transmission Unit`) — это максимальный размер пакета, который может быть передан через данный линк без фрагментации. Если размер IP-пакета превышает MTU следующего линка, пакет может быть разбит на несколько фрагментов.

При фрагментации в заголовке IP используются:

- **флаг `MF`** (`More Fragments`) — указывает, что за данным фрагментом следуют ещё фрагменты. Устанавливается в 1 для всех фрагментов, кроме последнего;
- **поле `Fragment Offset`** — указывает смещение данного фрагмента относительно начала исходного пакета, измеряется в единицах по 8 байт.

Фрагменты собираются обратно только на конечном узле-получателе. Промежуточные маршрутизаторы пересылают фрагменты независимо.

#### Флаг DF и ICMP-ошибка

Если на пакете установлен флаг **DF** (`Don't Fragment`), а MTU недостаточен для пересылки целого пакета, маршрутизатор не имеет права фрагментировать пакет. Вместо этого он:

1. отбрасывает пакет;
2. отправляет отправителю ICMP-сообщение `Destination Unreachable` с кодом `Fragmentation Needed` (тип 3, код 4);
3. в этом ICMP-сообщении указывает MTU следующего линка, что позволяет отправителю уменьшить размер пакетов.

Этот механизм лежит в основе **Path MTU Discovery** — алгоритма определения максимального размера пакета, который может быть доставлен до получателя без фрагментации.

#### Что необходимо понять по итогам теории

После изучения теоретического материала студент должен понимать:

1. как работает статическая маршрутизация;
2. почему маршрут `/24` имеет приоритет над `/16`;
3. зачем нужно поле TTL и как оно изменяется;
4. что такое MTU и когда возникает фрагментация;
5. что означают флаги MF и DF;
6. почему маршрутизатор отправляет ICMP `fragmentation needed`.

### Общие сведения о GNS3 и Linux-узлах

#### Среда GNS3

В лабораторной работе используются:

- `PC1`, `PC2` — Linux-узлы;
- `R1`, `R2`, `R3` — Linux-узлы с двумя сетевыми интерфейсами каждый.

Такая линейная топология удобна для демонстрации:

1. маршрутизации по нескольким переходам;
2. уменьшения TTL;
3. работы наиболее специфичного маршрута;
4. фрагментации на отдельном линке.

#### Команды настройки сети и маршрутизации

Для настройки IP-адресов и маршрутизации на Linux-узлах используются следующие команды:

| Команда | Назначение |
|---|---|
| `ip addr add <IP>/<маска> dev <интерфейс>` | назначить IP-адрес |
| `ip link set <интерфейс> up` | поднять интерфейс |
| `ip route add <сеть>/<маска> via <шлюз>` | добавить статический маршрут |
| `ip route del <сеть>/<маска>` | удалить маршрут |
| `ip route get <IP-адрес>` | показать, какой маршрут будет выбран |
| `ip route show` | показать таблицу маршрутов |
| `sysctl -w net.ipv4.ip_forward=1` | включить IP forwarding |
| `ip link set <интерфейс> mtu <значение>` | изменить MTU |
| `ping -s <размер> <IP-адрес>` | отправить ping с заданным размером payload |
| `ping -M do -s <размер> <IP-адрес>` | отправить ping с установленным флагом DF |
